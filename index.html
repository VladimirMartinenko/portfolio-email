Basic principles that I use when coding HTML email:

1.Table layout

This method allows letters to be displayed correctly on a wide variety of client software.

2.Inline CSS styles

Although using external CSS files is a common practice in web development, many email clients do not support referencing external styles. Therefore, it is recommended to use inline styles directly in HTML tags or in a <style> tag inside a <head> element. This allows you to control the visual presentation of the letter more reliably.

3.fonts

Using standard system fonts to display email texts is the simplest and most reliable solution. But if you need to make changes to the typography, you can use web fonts, such as Google Fonts.

4.Letter size

The recommended width of the content part is no more than 600 pixels. This size fits most devices and is easily adaptable. Anything outside the main container can be any size.

5.Adaptive layout

The recommended widt.In the case of multi-column emails, everything is much more complicated, since the columns that were located next to each other in the desktop version of the email must be redistributed to display it on a smartphone.

6.Testing emails

You can test emails in various ways: Send a letter to your email: desktop (Outlook), web client (Gmail) and mobile client (iOS Mail). Test automatically using Litmus or Email on Acid.
